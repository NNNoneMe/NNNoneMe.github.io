javaSE部分

1.简单讲一下java跨平台原理

由于各操作系统支持的指令集（或者是实现原理）不是完全一致的，就会让我们的程序在不同的的操作系统上执行不同的代码。java开发了使用与不同操作系统的java虚拟机来屏蔽各个系统之间的差异，提供了统一的接口，对于开发者而言，只要在不同的系统上安装对应的java虚拟机，编程时遵循java的规范，就可以实现在所有操作系统上运java程序了。

2.讲一下java中的int数据占据多少字节？（考察java基本数据类型）

java提供了8种基本数据类型：

整型：byte，short，int,long

浮点型：float,double

字符类型：char

布尔类型：boolean

int占32位，4个字节

3.面向对象的特征有哪方面（抽象问题，要举例说明）

封装：将 一个对象封装成一个高度自治和相对封闭的个体，对象的状态（属性）由该对象自己的行为（方法）来读取和改变。

继承：在定义和实现一个类的时候，可以在一个已经存在的类的基础上来进行，把这个已经存在的类所定义的内容作为自己的内容，并且可以加入新的内容，或者修改原来的方法使其更时候特殊的需求，这就是继承。

多态：多态是指程序中所定义的引用变量所指向的具体类型和通过该引用变量发出的方法调用在编程时并不确定，而是在程序运行时期才确定，即一个引用变量到底会指向哪个类的实例对象，该引用变量发出的方法调用到底是哪个类中实现的方法，必须在程序运行时期才能决定。

抽象：抽象就是找出一些事物的相似和共性之处，然后将它们归为一个类，该类只考虑这些事物的相似和共性，忽略与当前目标无关的方面。

4.有了基本的数据类型，为什么还需要包装类型？

装箱：把基本的数据类型转换为对应的包装类型

拆箱：把包装类型转换为基本数据类型

java是一个面向对象的语言，而基本数据类型不具备面向对象的特性。

举例：分别用Integer和int来表示Person类的ID，根据ID判断是否存在这个person时，包装类型只需要判断是否为空就行了

7.讲一下java中的集合

java中的集合分为value，key-value两种（Conllection,Map)

存储值的有List和Set；存key-value的有Map；

List是有序的，可以重复的；Set是无序的，不可以重复（根据equals和hashCode判断，对象如果要存在Set中，必须重写equals方法和hashCode方法。）

8.ArrayList和LinkedList的区别

List常用的有ArrayList和LinkedList。

ArrayList底层时数组实现，查询效率高，插入删除效率低；LinkedList使用的是链表结构，查询慢，删除插入效率高；故而前者用在查询比较多，插入和删除比较少的情况，后者适用于删除、插入多，查询相对较少的情况。

数组具有索引，查询特定的元素比较快，而插入、删除、修改比较快，数组在内存中是一块连续的内存，插入或者删除时，需要移动内存。

链表不要求内存是连续的，查询比较慢，插入、删除效率高。在当前元素中存放上一个或下一个元素的地址，查询时需要从头部开始，效率较低。插入时不需要移动内存，只需要改变引用指向即可。

9.讲一下HashMap和HashTable的区别？HashTable和CurrentMap的区别？

相同点：HashMap和HashTable都可以来存相互key-value的数据。

1.HashMap是可以把null作为key或value的，HashTable不行；

2.HashMap线程不安，效率高；HashTable线程安全，效率较低；

想线程安全，又想效率高，怎么办？CurrentMap（JDK1.5)

通过把整个HashMap分为N个Segment（类似HashTable），可以提供相同的线程安全，效率却比HashMap提升了N倍。

javaWeb部分

1.讲一下http GET和POST请求的区别？

GET和POST请求都是http的提交方式，用户通过不同http的提交方式完成对资源的操作。具体来讲GET一般用于查询/获取资源信息，而POST一般用于更新资源信息。

http定义了与服务器交互的不同方式，基本的方法有4中，分别就是GET,POST,PUT,DELET它们对应着对信息资源的查、改、增、删4个操作。

1.GET请求提交的数据会在地址栏显示出来，而POST请求不会在地址栏显示出来。

GET提交，请求数据会附加在URL之后，以?分割URL和传输数据，多个参数用&连接;POST提交：把提交的数据防止在http请求包的包体中。

2.数据传输的大小

在http协议中，GET请求由于浏览器对URL长度的限制导致传输的数据有限制，而POST请求没有这一限制。

3.安全性

POST提交方式的安全性比GET提交方式高。因为GET的数据是在地址中显示出来的，可以通过历史记录来找到密码等敏感信息。

2.说一下你对servlet的理解？

是用java编写的服务器端程序。这些servlet都要事先Servlet接口。其主要功能是在于交互式的浏览和修改数据，生成动态Web内容。

HttpServlet重写doGet和doPost方法或者重写service方法完成催get和post请求的响应。

3.简单说一下servlet的生命周期？

servlet具有良好的生存周期的定义，包括加载和实例化、初始化、处理请求以及服务结束。这个生成周期由Servlet接口中的init,service和destroy方法表达。

Servlet启动时，Servlet生命周期开始。Servlet被服务器实例化后，容器运行其init方法，请求到达时运行其service方法，service方法自动派遣运行与请求对应的doGet/doPost方法等，当服务器销毁Servlet实例的时候或者服务器关闭调用destroy方法。

具体流程：

加载Servlet的class–>实例化Servlet–>调用Servlet的init方法–>响应请求（service方法处理）–>Servlet容器关闭（destroy方法）

4.Servlet中forward()和redirect()的区别？

前者仅是容器中控制权的转向，在客户端浏览器地址中不会显示转向后的地址；后者则是跳转，浏览器将会得到跳转的地址，并且重新发送请求，从浏览器地址可以看到跳转后的地址。所以前者更加高效。而且利于隐藏实际链接。但是在有些情况下，需要跳转到其他服务器上的资源，必须用后者。

7.说一下session和cookie的区别

session和cookie都是会话跟踪技术。cookie数通过在客户端记录信息确定用户身份，session通过在服务器端记录信息确定用户身份。但是session的实现依赖于cookie（sessionId是session的唯一标识，需要存放在客户端）

cookie和session的区别：

1.cookie数据存放在客户的浏览器上，session数据存放在服务器中。

2.cookie不是很安全，别人可以分析存放在本地的cookie，进行cookie欺骗。

3.减轻服服务器性能，可以使用cookie。

4.单个cookie保存的数据不能超过4K,很多浏览器都限制一个站点最多保存20个cookie

购物车的实现最好使用cookie，但是cookie是可以在客户端禁用的，这时候，我们要使用cookie+数据库的方式实现，当从cookie中不能取出数据时，就从数据库中获取

2.简单介绍一下关系型数据库的三大范式？

范式就是规范，关系型数据库在设计表时，需要遵循的三个规范。

第一范式：指数据表中的每一列都是不可分割的基本数据项，同一列中不能有多个值，即实体中的某个属性不能有多个值或者不能有重复的属性。列的数据不可分割。

第二范式：要求数据表中的每一行必须可以被唯一的区分。为实现区分通常需要为表加上一个列，用以存储各个实例的唯一标识（主键）。

第三范式：要求一个数据表中不能包含已经在其他表中已经包含的非主键关键字信息（外键）。

3.事务四个基本特性

事务是并发控制的单位，用户定义一个操作序列。这些操作要么都做，要么都不做，是一个不可分割的工作单位。

比如：一个转账必须A账号扣钱成功，B账号加钱成功，才算真正的转账成功。

事务必须满足四大特征：原子性，一致性，隔离性，持久性

原子性：表示事务内部不可分割，要么都成功，要么都失败。

一致性：要么都成功，要么都失败，失败后要对前面的操作进行回滚。

隔离性：一个事务开始后，不能被其他事务干扰。

持久性：事务开始了，就不能终止。

触发器，需要有触发条件，当条件满足后做什么操作。

触发器的用处有很多，比如一些网站，发布一个日志，自动通知好友，其实就是在增加日志时做一个后触发，再向通知表中写入条目。

6.简单讲一下数据库的存储过程的使用场景

数据库存储过程具有如下有点：

1.存储过程只在创建时进行编译，以后每次执行存储过程都不需要重新编译，而一般的sql语句每执行一次就编译一次，因此使用存储过程可以大大提高数据库执行速度。

2.通常，复杂的业务逻辑需要多条sql语句。这些语句要分别从客户机发送到服务器，当客户机和服务器之间的操作很多时，将会产生大量的网络传输。如果将这些操作放在一个存储过程中，那么客户机和服务器之间的网络传输就会大大减少，降低了网络负载。

3.存储过程创建一次便可以重复使用，从而减少数据库开发人员的工作量。

4.安全性高，存储过程可以屏蔽对底层数据库对象的直接访问，使用EXCUTE权限调用存储过程，无需拥有访问底层数据库对象的显示权限。

7.说说你对jdbc的理解

java database connection 数据库连接，数据库管理系统有很多，每个数据库管理系统支持的命令不是一样的。

java中只定义了接口，让数据库厂商自己实现接口，对我们而言，只需要导入对应厂商开发的实现即可。

8.JDBC中的PreparedStatement相比Statement的好处

大多情况下我们都是用PreparedStatement代替Statement，理由如下

1.PreparedStatement是预编译的，比Statement速度快；

2.代码的可读性和可维护性更高，可以使用占位符；

3.安全性：PreparedStatement可以繁殖SQL注入攻击，而Statement却不能。

9.数据库连接池的作用

1.限定数据库连接的个数，不会导致由于数据库连接过多而导致系统运行缓慢或者崩溃；

2.数据库连接不不需要每次都去创建或者销毁，为我们节约了资源；

3.数据库连接不需要每次都去创建，响应时间更快
